# Безопасность

## Сообщение об уязвимостях

Если вы обнаружили уязвимость в проекте, пожалуйста, **НЕ** создавайте публичный Issue.

### Как сообщить

1. Отправьте email на: **security@obrazz.app**
2. Включите в сообщение:
   - Описание уязвимости
   - Шаги для воспроизведения
   - Потенциальное влияние
   - Предложения по исправлению (если есть)

### Время ответа

- Подтверждение получения: в течение 48 часов
- Первоначальная оценка: в течение 7 дней
- Исправление: зависит от серьёзности

## Политика безопасности

### Поддерживаемые версии

| Версия | Поддержка |
|--------|-----------|
| 1.x.x  | ✅ Активная |
| < 1.0  | ❌ Не поддерживается |

### Лучшие практики

#### Клиент
- Не храните секреты в клиентском коде
- Используйте HTTPS
- Валидируйте пользовательский ввод

#### Сервер
- Используйте переменные окружения для секретов
- Валидируйте входные данные через Zod
- Используйте параметризованные запросы (Drizzle ORM)
- Настройте CORS правильно
- Добавьте rate limiting для API

#### База данных
- Используйте SSL соединения
- Регулярно обновляйте пароли
- Ограничьте привилегии пользователя БД

## Зависимости

Регулярно проверяйте зависимости на уязвимости:

```bash
# Аудит npm пакетов
npm audit

# Исправление уязвимостей
npm audit fix
```
